# Diagnostic Task: Fix Gradient Explosion from Pareto Heavy-Tail

## 🚨 Observed Phenomenon
Training stops abruptly at Episode 6 with an Episode Reward of -2,165,835. The `Episode Q0` successfully updated (not zero anymore), but the massive negative reward from the Pareto heavy-tailed distribution caused a gradient explosion and `NaN` weights, triggering MATLAB's failsafe stop.

## 🎯 Task Overview
Cap the maximum file size generated by the FTP3 Pareto distribution to a realistic physical limit (e.g., 200 Mbits) and clip the maximum single-step reward penalty to prevent neural network divergence.

---

## 🟢 Phase 1: Cap Pareto File Size in `RANSlicingEnv.m`
**Goal: Prevent the heavy-tailed distribution from generating astronomically large files.**

- [ ] **Step 1.1: Modify `sampleParetoBits`**
  - Open `RANSlicingEnv.m` and locate the `sampleParetoBits` method.
  - After `sample = xm / (uniformSample ^ (1 / alpha));`, add a hard cap:
    ```matlab
    % Cap the maximum file size to 200 Mbits (~25 MB) to prevent numerical explosion
    sample = min(sample, 200e6);
    ```

## 🟢 Phase 2: Clip Reward Penalty in `calculateReward.m`
**Goal: Provide a safety net for the Critic network to avoid exploding gradients.**

- [ ] **Step 2.1: Modify `embbQueuePenalty`**
  - Open `calculateReward.m`.
  - Locate `embbQueuePenalty = -Config.embb_penalty_scale * sum(queueBits);`.
  - Add a clipping function to limit the single-step penalty to -50.0:
    ```matlab
    embbQueuePenalty = -Config.embb_penalty_scale * sum(queueBits);
    embbQueuePenalty = max(-50.0, embbQueuePenalty); % Clip massive negative rewards
    ```
